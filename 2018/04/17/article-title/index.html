用github和Hexo框架搭建免费个人博客

2018-04-17 • 标签: github Hexo

早就有搭一个自己的博客想法，最近终于下定决心花一天时间搭建了这个个人博客。考虑了许多方案，最后决定采用Hexo框架，并将其部署在github page。Hexo 是一个快速、简洁且高效的博客框架。 Hexo 使用 Markdown解析文章，在几秒内，即可利用靓丽的主题生成静态网页。github page是一个静态的网站托管服务，旨在直接从GitHub库中托管您的个人、组织或项目页面。

准备工作

想要在github上部署网页，需要要有github账号，并在PC上安装npm、git和node.js，安装教程网上有很多，这里就不一一赘述了。

设置github page

 登录了github之后，新建一个仓库，并以yourgithub.github.io命名。见下图(因为我已经有这个命名仓库，所以出现红色叹号)：
![Create Repository](article-title/01.png)

点进新建的仓库，点击Settings按钮，拉到下面。可看到如下画面：

可以看到绿色背景部分，说明github page已经部署好，点击蓝色链接可以访问。
部署好github page 之后，可以将github 部分先放一边。接下来进行Hexo框架的设置。

安装Hexo

首先在你喜欢的地方创建一个文件夹（你以后写博客的时候需要在这个文件夹中操作），然后通过命令行进入这个文件夹，并在这个文件中安装Hexo。

安装完成后可以输入hexo -v检查时候安装成功

D:\blog>hexo -v

安装成功后，输入hexo 是新建一个网站，所以可能需要花一些时间.请耐心等待。
D:\blog>hexo init

初始化完成后，输入npm install，安装所需要的组件，结束后输入hexo g，生成静态文件；然后输入hexo s启动服务器。

``
D:\blog>npm install
D:\blog>hexo g
D:\blog>hexo s


接下来到了激动人心的时候，复制上图中的链接在浏览器中打开可看到下面的画面，说明Hexo在本地服务器部署成功。如果页面一直无法跳转，那么可能是端口被占用了。输入ctrl+c停止服务器，然后输入“hexo server -p 端口号”来改变端口号吧。

部署到github page
