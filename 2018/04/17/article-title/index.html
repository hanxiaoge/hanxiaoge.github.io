用github和Hexo框架搭建免费个人博客

2018-04-17 • 标签: github Hexo

早就有搭一个自己的博客想法，最近终于下定决心花一天时间搭建了这个个人博客。考虑了许多方案，最后决定采用Hexo框架，并将其部署在github page。Hexo 是一个快速、简洁且高效的博客框架。 Hexo 使用 Markdown解析文章，在几秒内，即可利用靓丽的主题生成静态网页。github page是一个静态的网站托管服务，旨在直接从GitHub库中托管您的个人、组织或项目页面。

准备工作

想要在github上部署网页，需要要有github账号，并在PC上安装npm、git和node.js，安装教程网上有很多，这里就不一一赘述了。

设置github page

登录了github之后，新建一个仓库，并以yourgithub.github.io命名。见下图(因为我已经有这个命名仓库，所以出现红色叹号)：

点进新建的仓库，点击Settings按钮，拉到下面。可看到如下画面：

可以看到绿色背景部分，说明github page已经部署好，点击蓝色链接可以访问。
部署好github page 之后，可以将github 部分先放一边。接下来进行Hexo框架的设置。

安装Hexo

首先在你喜欢的地方创建一个文件夹（你以后写博客的时候需要在这个文件夹中操作），然后通过命令行进入这个文件夹，并在这个文件中安装Hexo。

安装完成后可以输入hexo -v检查是否安装成功

安装成功后，输入hexo 是新建一个网站，所以可能需要花一些时间.请耐心等待。

初始化完成后，输入npm install，安装所需要的组件，结束后输入hexo g，生成静态文件；然后输入hexo s启动服务器。

接下来到了激动人心的时候，复制上图中的链接在浏览器中打开可看到下面的画面，说明Hexo在本地服务器部署成功。如果页面一直无法跳转，那么可能是端口被占用了。输入ctrl+c停止服务器，然后输入hexo server -p端口号”来改变端口号吧。

这个时候你就可以新建自己的博客了。在工作台执行命令 hexo new post "article-title" 便可以新建一篇博客。这时候，你可以在文件夹_posts目录下找到你创建的博客文件，然后可以用Markdown语法规则编写博客。

你新建的博客文档是没有文件夹的，这个文件夹是用来盛放博客中的图片文件的，如果你有需要你可安装一个扩展：npm install hexo-asset-image --save，安装完成后新建的博客就有文件夹了，之前创建的博文可以手动添加文件夹。
此时，运行服务器hexo s,刷新刚才的网页，便可看到我们创建的博文。

如果你已经配置了github 的SSH秘钥，可以本地操作远程库了，就可以进行下面的步骤了（如果没有配置，就先进行配置，配置教程网上很多）。首先配置Deployment，在我们安装Hexo的文件夹中找到_config.yml文件，修改deploy属性（在下方）。

在部署之前还需要添加一个用于在github上部署网站的扩展，其命令为：npm install hexo-deployer-git --save ,添加完成后
运行hexo d -g,完成后便把Hexo站点部署到了github上面。

更换默认主题

部署完成后的网站页面是Hexo框架的默认主题，如果你不喜欢，可以更换掉默认主题，Hexo官网有主题专栏,也可以在网上找你喜欢的开源主题。本博客采用的是Huno主题，在此感谢作者letiantian （该主题下的评论功能扩展和mathjax公式支持已经不能正常时候了,请选择该主题的同学自行寻找代替方案），找到主题后下载并保存到themes文件夹下：

下载完成后，修改主文件夹下的_config.yml配置文件，将themes属性修改为你选择的主题名：themes：yourthemes

在操作台运行 hexo s，查看你的新主题。

主题配置

主题的布局可能不符合你的需求，这时候就需要你按照主题的说明文件对主题进行调整，此时主要修改主题文件夹下的_config.yml配置文件，具体请参考主题说明文档。

后记

了解一下Hexo框架的相关知识和markdown的语法结构对主题的修改后后期的博文写作很有帮助，建议大家动手之前和操作中多查看。另外，如果你觉的这篇博文对你有帮助，不妨带个赞再走，如有问题可留言。
